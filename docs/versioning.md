# Semantic Versioning Strategy

This project follows **SemVer 2.0.0**.

- **MAJOR**: incompatible script behavior changes
- **MINOR**: backward-compatible new tools/features
- **PATCH**: backward-compatible fixes and documentation updates

Examples:
- `1.0.0` initial stable toolkit
- `1.1.0` adds new diagnostics modules
- `1.1.1` fixes a logging bug
