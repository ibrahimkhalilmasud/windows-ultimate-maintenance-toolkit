# Security Policy

## Supported Versions
| Version | Supported |
|---------|-----------|
| 1.x     | ✅ |
| < 1.0   | ❌ |

## Reporting a Vulnerability
Please create a private security advisory in GitHub or email the maintainer listed in the repository profile.

Include:
- Affected script(s)
- Reproduction steps
- Potential impact
- Suggested remediation (if available)

## Security Principles for This Toolkit
- Never permanently disable endpoint security controls.
- Never modify activation or licensing components.
- Perform only validated cleanup and repair operations.
- Keep privileged operations explicit and logged.
